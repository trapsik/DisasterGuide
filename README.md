# 🚨 재난 가이드 (Disaster Guide)
> **공공데이터와 실시간 위치 정보를 결합한 안드로이드 기반 통합 재난 대응 플랫폼**

---

## 📌 프로젝트 개요
* **개발 기간:** 2023.09 - 2023.12 (3인 팀 프로젝트)
* **주요 타겟:** 재난 상황 시 파편화된 안전 정보를 신속하게 확인하고, 지인의 안전(위치)을 실시간으로 파악하고자 하는 사용자
* **핵심 가치:** 사후 대응을 넘어 기상 예보를 통한 사전 예방, 위치 기반 대피소 탐색, 사용자 간 현장 정보 공유를 아우르는 통합 안전망 구축

---

## 🛠️ 사용 기술 스택 (Tech Stack)

### Client (Frontend)
* **Language:** Java 1.8
* **Framework/SDK:** Android SDK (API 33), Google Maps SDK for Android
* **Library:** Jackson-databind (JSON/CSV Parsing)

### Server (Backend) & Database
* **Architecture:** Spring Boot 기반 RESTful API 서버 (팀원 협업 및 연동)
* **Database:** MySQL (ERD 구조 분석 및 데이터 무결성 검증 참여)

---

## ✨ 핵심 기능 (Key Features)

1. **위치 기반 대피 시설 매핑 (GIS)**
   * Google Maps API를 연동하여 사용자의 현재 위치를 실시간으로 추적합니다.
   * 공공데이터포털의 대피 시설 데이터(민방위 대피소, 지진 대피소, 응급실)를 파싱하여 지도 위에 직관적인 마커로 시각화합니다.
2. **소셜 안전 네트워크 (세이프티 트래킹)**
   * 지인 간 친구 추가 및 실시간 위치 공유 기능을 제공합니다.
   * 비상 상황 시 가족 및 지인의 최종 위경도 좌표를 지도상에 렌더링하여 안전 여부를 확인할 수 있도록 돕습니다.
3. **양방향 재난 커뮤니티**
   * 현장 상황을 사용자가 직접 제보하고 공유할 수 있는 게시판 및 댓글 시스템을 지원합니다.
4. **실시간 기상 대시보드**
   * 기상 API 및 날씨 컨트롤러 연동을 통해 현재 기상 특보 및 재난 정보를 메인 화면에 상시 노출합니다.

---

## 🏗️ 설계 및 구현 특징 (Engineering Focus)

* **데이터-UI 관심사 분리 (Separation of Concerns):**
  * 10여 종의 독립적인 도메인 모델(`UserModel`, `PostModel`, `LocationModel` 등)과 UI 렌더링 로직을 엄격히 분리했습니다.
  * 게시글, 댓글, 친구 요청 등 각기 다른 도메인 데이터에 최적화된 **7종의 커스텀 어댑터(`ListViewAdapter`)**를 직접 구현하여 복합 데이터를 안정적으로 핸들링했습니다.
* **E2E 통합 테스트 및 API 검증:**
  * 백엔드 Spring Boot 서버와의 연동 과정에서 회원가입 포맷(정규식 제한 조건), 로그인 세션 관리 등 총 9개 도메인 API의 예외 처리 상태를 전수 검증했습니다.
  * 커뮤니티 데이터 업로드 시 데이터베이스 내 **외래키(FK) 참조 무결성 제약 조건**이 깨지지 않도록 예외 처리 로직을 검토하고 보완했습니다.

## 🚀 트러블 슈팅 (Technical Challenges)

### 1. 클라이언트-서버 간 예외 처리 미흡으로 인한 앱 안정성 이슈
* **문제:** 삭제된 게시글에 댓글을 달거나 존재하지 않는 ID로 데이터 업로드 요청 시, 서버 내부 SQL 에러(외래키 무결성 제약 위반)로 인해 앱이 비정상 종료되는 현상 발생.
* **해결:** 백엔드 개발자와 협의하여 서버 측에서 데이터 무결성 예외를 캐치하도록 수정하고, 규격화된 실패 응답(`ResponseModel`)을 반환하도록 설계 변경. 클라이언트 단에서는 이 응답 코드를 분기 처리하여 사용자에게 명확한 예외 안내 팝업을 제공함으로써 앱의 안정성을 확보했습니다.

### 2. 비동기 데이터 통신과 UI 상태 불일치
* **문제:** 친구 요청 수락(`friend/accept`) 또는 삭제 시, DB 내의 상태 값은 정상 변경되나 화면상의 리스트 뷰가 즉각적으로 갱신되지 않는 문제 발생.
* **해결:** 서버 통신 성공 콜백 시점에 어댑터 내부의 데이터셋을 직접 동기화한 후, `notifyDataSetChanged()`를 명시적으로 호출하여 별도의 새로고침 없이도 즉각적으로 UI 갱신이 이루어지도록 해결했습니다.
